Understanding these real problems explains why we built what we built

Challenge 1: Large Structured Data

Problem: Excel files with 1000s of rows, participant databases with complex relationships

Why LLMs failed: Context windows too small, auto-regressive models terrible at tabular data, hallucinations on counts/aggregations

Example: "What's the average age for PVT rank soldiers?" - LLM would hallucinate numbers

→ Solution: SQL Code Generator Agent

Challenge 2: Complex Military Documents

Problem: Military documents with diagrams, tables, sophisticated layouts that text parsers missed

Why text extraction failed: Lost 60% of information in tables/diagrams, no context from visual elements

Example: Network architecture diagrams came out as gibberish text

→ Solution: Multimodal Document Processing

Challenge 3: Poor Retrieval Accuracy

Problem: Pure semantic search missed exact terms, keyword search missed similar concepts

Why single approach failed: Military docs need both exact match (acronyms) and semantic understanding

Example: Searching "UAV" wouldn't find "unmanned aerial vehicle" and vice versa

→ Solution: Hybrid Search (15% improvement)

Challenge 4: Dynamic Document Questions

Problem: 15 different questions per document type, needed audit trail for decisions

Why static approach failed: Context limits with mini-RAG needed, reasoning transparency required

Example: Extract specific compliance items from 100-page requirements docs

→ Solution: Q&A Agent with Mini-RAG
